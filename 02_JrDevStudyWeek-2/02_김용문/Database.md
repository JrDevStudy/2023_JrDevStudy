# DATABASE 개념 및 종류

## DATABASE란?
> - 한마디로 정의하면 `데이터의 집합` 이라고 하며 짧게 DB라고 한다.
> - DB에는 일상생활 대부분의 정보가 저장되고 관리됩니다.

## SQL이란
SQL은 관계형 데이터베이스 관리 시스템(RDBMS)의 데이터를 관리하기 위해 설계된 특수 목적의 프로그래밍 언어라고한다.

## 관계형 데이터베이스(RDB)란?
> - 현재 가장 많이 사용되는 데이터베이스로서 관계형 데이터 모델을 기초로 두고 모든 데이터를 2차원 테이블 형태로 표현하는 데이터베이스이다. 
> - 흔히 행(Column)과 열(Row)로 표현되는 테이블간의 관계를 나타낼 때 사용되며, 이렇게 표현된 데이터 베이스는 SQL을 통하여 관리 및 접근한다.
> - 구성된 테이블이 다른 테이블들과 관계를 맺고 모여잇는 집합체 이고 이러한 관계를 나타내기 위해 외래 키(Foreign Key)를 사용하며 외래 키를 이용한 테이블 간 Join이 가능하다는게 가장 큰 특징이다.

## RDB의 종류

Oracle, MySQL(Oracle) / MS-SQL(Microsoft) / DB2, Infomix(IBM) / Maria DB(Sun Microsystems) / Derby(Apache) / SQLite(Opensource)

## Oracle 
> - 1978년 로랜스 J. 엘리슨이 오라클 첫번째 버전을 개발하였다.
> - 미국 오라클사의 관계형 데이터베이스 관리 시스템(RDBMS)이다.
> - 오픈 소스 DBMS가 있음에도 안정성과 유지보수를 보장받을 수 있다는 장점 때문에 비 IT업종 기업에서 많이 사용한다.

### Oracle 특징
> - 기업용으로 주로 사용하며, 중견기업, 대기업 등 대량의 정보관리하기에 최적화 되어있다.
> - Unix, Linux, Windows 등의 대부분의 OS를 지원하고 있다.
> - 오라클 자체 SQL 쿼리를 사용해 표준 형식과 약간 다르다.

## MS-SQL
> - Microsoft에서 나온 RDB이며 사용자의 편의를 위해서 좀 더 사용이 간편하다.

### MS-SQL 특징
> - 윈도우 개발 환경에서 DB가 필요할 때 MS-SQL을 많이 사용하며 .net, VS와 쉽게 연동이 가능하다.
> - Oracle에 비해 비용이 저렴하지만 그래도 사용량이 적다.

## MySQL
> - 1990년대 MySQL AB 사에 의해 첫번쨰 버전이 발표 되었고 2008년에 Sun 마이크로시스템즈사가 인수하고 2009년에 Oracle에서 Sun을 인수함에 따라 현재는 Oracle의 소유이다.

### MySQL 특징
> - Oracle과 동일하게 Unix, Linux, Windows 등의 대부분의 OS를 지원하고 있다.
> - 오픈소스의 장점으로 많은 기업에서 홈페이지나 쇼핑몰 등 일반적인 웹 개발에 많이 이용되고 있습니다.
> - 오라클에 비해 사용자 편의를 위한 기능, 사용자 실수 또는 재해에 대비한 기능, 성능성 향상 등 기능적인 한계가 있다.
> - 다만 오픈소스이다 보니 많은 기업에서 활용되고 있다.

## Maria DB
> - 2009년에 발표되었으며 Oracle이 MySQL을 인수하면서 Oracle을 더 많이 판매하기 위해서 오픈소스인 MySQL의 기능을 감소 시켰는데 이에 MySQL을 개발한 마이클 몬티 와이드니어스는 Oracle과의 의견 충돌로 회사를 나와 새롭게 개발했다고 한다.

### Maria DB 특징
> - MySQL과 완벽하게 호환되며 기본적인 명령어나 사용방법까지 동일하기 때문에 개발자 또는 DBA가 쉽게 접할수 있다.
> - MySQL에 비해 성능적인 부분에서는 향상 되었기 때문에 최근 사용하는 기업이 늘어나는 추세이다.

## DB2
> - 대형화된 데이터 관리를 목적으로 만들어진 IBM의 관계형 데이터베이스 관리 시스템이다.
> - 1983년에 발표되었으며, 사용자들이 서로 관계된 여러 개의 데이터베이스를 동시에 접근할 수 있다.

### DB2 특징
> - 각 업무의 특성에 맞게 시스템이 최적화 될수 있으며, 자기 최적화, 자가 치유, 자가 구성, 워크로드 관리, 확장된 자동화 기능 등 다양한 기능을 구현할수 있다.
> - 데이터 압축 기술이 좋기 때문에 대형화된 데이터를 다루는데 최적화 될 수 있다는 장점이 있어 많은 중견기업, 대기업 권에서 DB2를 활용하고 있다.

## Sybase
> - 1984년에 공개한 관계형 데이터베이스이며 2010년에 SAP에서 인수했다.

### Sybase 특징
> - 하나의 DBMS에 여러개의 DB를 생성할수 있고 Oracle에 비해 비용이 저렴하여 대기업에서 많이 사용할것 같지만 아직까지는 Oracle이 더 많이 사용된다.

## NoSQL란?
> - RDB의 특성 뿐만 아니라 다른 특성까지도 지원해 주는 Databse라는 의미를 지닌다.
> - 대용량 데이터를 다루거나 데이터 분산 처리에 용이 및 유연한 데이터 모델링이 가능하다는 특징이 있다.
> - 여러가지 방식이 있어서 사용할 방향에 따라 방식을 정하는 것이 좋다.

### NoSQL 방식
> - Document 방식, Key-Value 방식, Big Table DB가 대표적이다.

1. Document 방식
> - Json, XML과 같은 Collection 데이터 모델 구조를 사용한다.
> - 대표적으로 MongoDB를 많이 사용합니다.
2. Key-Value 방식
> - Key와 Value의 데이터가 쌍으로 저장되는 가장 단순한 데이터 모델 구조 사용
> - 대표적으로는 Redis를 많이 사용합니다.
